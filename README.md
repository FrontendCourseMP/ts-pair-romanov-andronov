## Лабораторная работа 1: TimeParser (Калькулятор времени поезда)
Приложение для расчета нового времени прибытия поезда с учетом задержки.
Точка входа для Zod:
Импортируется библиотека Zod в файле `src/labs/TimeParser.ts` для валидации входных данных формы.

Основные объекты и методы Zod:
z.string() - валидация строковых значений
z.number() - валидация числовых значений  
.min() / .max() - ограничение диапазона значений (0-23 для часов)
.regex() - проверка формата строки (для времени)
.refine() - кастомная валидация
.parse() / .safeParse() - методы валидации с обработкой ошибок

Типы данных, передаваемые в функции/методы
Входные параметры:
scheduledTime: string - время по расписанию в формате ЧЧ (строка)
hoursDelay: number - количество часов задержки (число от 0)

Возвращаемое значение:
string - новое время прибытия в формате ЧЧ

Интерфейсы:

interface Time {
  hours: number
}

interface TimeCalculator {
  calculateNewArrivalTime(scheduledTime: string, hoursDelay: number): string
}

Функционал
- Валидация времени по расписанию (0-23 часа)
- Валидация количества часов задержки (неотрицательное число)
- Расчет нового времени с учетом 24-часового формата
- Обработка ошибок и вывод понятных сообщений пользователю

## Лабораторная работа 2: Brackets (Проверка парности скобок)
Приложение для проверки правильности расстановки парных скобок в строке.
Точка входа для Zod:
Импортируется библиотека Zod в файле `src/brackets/handleSubmit.ts` для валидации входной строки.

Основные объекты и методы Zod:
z.string() - валидация строки со скобками
.regex() - проверка, что строка содержит только допустимые символы: `()`, `[]`, `{}`
.min() - проверка минимальной длины (строка не пустая)
.transform() - преобразование данных
.safeParse() - безопасная валидация с обработкой ошибок

Входные параметры:
input: string - строка со скобками для проверки

Возвращаемое значение:
-boolean - true если скобки парные и правильно расставлены, `false` в противном случае

Вспомогательные структуры:

const pairs: { [key: string]: string } = {
  "(": ")",
  "[": "]",
  "{": "}"
}

Функционал
- Валидация входной строки (только скобки: `()`, `[]`, `{}`)
- Проверка на пустую строку
- Алгоритм проверки парности с использованием стека
- Вывод результата валидации с указанием введенной строки

Модульная структура
isValidBrackets.ts - основная логика валидации скобок
handleSubmit.ts - обработчик отправки формы и интеграция с DOM
brackets.ts- подключение обработчика события к форме
